About this Application

This is an application submission for Riot's First API Challenge. It is a two week challenge that involves using game data from U.R.F matches and using various endpoints provided by Riot to create something (anything!).

The purpose of this application is to have a high level overview of URF game data that is easily readable by us humans. The design used the original api-match endpoint to feed match ids into the application's own database. From there, a utility was used to fetch all the match data from the match endpoint and dissect and populate into collections I thought would be best for the purpose of this app. ALL the data is based on what is in the application's database, not Riot's. You may experience some "lag" inbetween loading certain pages, especially the Players' page. This is because all the information you see is dynamic and live! That means if the database got updated, the page would update instantaneously too. As more matches are loaded into the database, more documents (rows) need to be processed. Think of it this way though: see the number on the front page/dashboard for Total Matches Played? Well all the data you see on this site is from that many number of matches! ;)

Technically, this web application uses a MEAN stack and comprises of both a REST back end application and an Angular front end application using Bootstrap. I followed the MEANJS example and highly encourage those interested in full webstack development in JS to "check it out" ;) . The application's database is hosted externally via MongoLab and I highly recommend them as well.

Technologies Used:

  • MongoDB
  • ExpressJS
  • AngularJS
  • NodeJS
  • Bootstrap
  • MongooseJS
